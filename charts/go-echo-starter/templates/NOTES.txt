2. Health endpoints:
   /api/v1/livez   — liveness (always 200 once the process is up)
   /api/v1/readyz  — readiness (aggregates registered health checks; goes 503 during graceful shutdown)
   /api/v1/healthz — alias of /api/v1/readyz

3. Supplying configuration via Kubernetes Secrets:
   The chart consumes user-supplied Secrets and ConfigMaps via the .Values.envFrom list. Reference any
   pre-existing resource without the chart owning it:

       envFrom:
         - secretRef:
             name: my-app-secrets       # APP_DATABASE_DSN, APP_OAUTH_*, etc.
         - configMapRef:
             name: my-app-config

   For TLS, set tls.enabled=true and tls.existingSecret to the name of a Secret with tls.crt / tls.key.
   The chart then mounts the cert at {{ .Values.tls.mountPath }} and points the daemon there via
   APP_SERVER_HTTPS_CERTIFICATE / APP_SERVER_HTTPS_KEY.

4. Seccomp profile:
   The chart defaults to seccompProfile.type=RuntimeDefault at both pod and container scope. To use a
   custom Localhost profile, set localhostProfile to a path under /var/lib/kubelet/seccomp/ that already
   exists on every node — the chart cannot ship the profile JSON itself.

5. Optional observability resources (default-off):
   metrics.serviceMonitor.enabled — Prometheus Operator ServiceMonitor (monitoring.coreos.com/v1)
   metrics.podMonitor.enabled     — Prometheus Operator PodMonitor    (monitoring.coreos.com/v1)
   logging.podLogs.enabled        — Grafana Alloy PodLogs              (monitoring.grafana.com/v1alpha2)

6. Run the bundled smoke test:
